## VuePress介绍

VuePress由两个部分组成：第一个部分是 **极简静态生成器**，它包含由Vue驱动的主题系统和插件API，另一个部分是为书写技术文档而优化的默认主题，它诞生的初衷是为了支持Vue及其子项目的文档需求。

## 废话不说，开始

1. 创建并进入一个新的目录

   ``` 
   mkdir blog-vuepress && cd blog-vuepress
   ```

2. 初始化。只会用`npm`，所以接下来就用`npm`命令。

   ```
   npm init
   ```

3. 将VuePress安装为本地依赖

   ```
   npm install -D vuepress 
   cnpm install -D vuepress # npm太慢了的话用这个命令，cnpm 需要 自行安装
   ```
```
   
{%note primary%}
   **npm install 命令的一些注意地方：**
   
npm install -g 全局安装（全局命令行使用）
   npm install -S 本地安装 即  npm install module_name --save  写入dependencies（将安装包放在./node_modules/）

   npm install -D 本地安装 即  npm install module_name --save-dev  写入devdependencies（将安装包放在./node_modules/）

   **dependencies与devDependencies有什么区别呢？**
   devDependencies 里面的插件只用于开发环境，不用于生产环境
   

dependencies 是需要发布到生产环境的
   **什么是开发环境什么是生产环境？**
   **开发环境**是开发项目时候要用到的程序，实际程序发布运行不一定需要。
   **生产环境**是程序运行过程中要运行的库。
   例如：gulp是用来压缩代码的工具，程序实际运行不需要用到，所以应该放在dev（开发环境）里。

   {%endnote%}

4. 创建第一篇文档

```
   mkdir docs && echo '# Hello VuePress' > docs\README.md
   ```

5. 在`package.json`中添加一些scripts.

   ```[] package.json
   {
     "scripts": {
       "docs:dev": "vuepress dev docs",
       "docs:build": "vuepress build docs"
     }
   }
   ```

6. 本地启动服务器

   ```
   npm run docs:dev
   ```

7. vuepress 简单的文档 基本搭建成功。
