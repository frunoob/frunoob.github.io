# 添加渐进式Web应用PWA支持

介绍

PWA（Progressive Web Apps，渐进式 Web 应用）运用现代的 Web API 以及传统的渐进式增强策略来创建跨平台 Web 应用程序。这些应用无处不在、功能丰富，使其具有与原生应用相同的用户体验优势。

# Install

yarn add -D @vuepress/plugin-pwa
# OR npm install -D @vuepress/plugin-pwa

# Usage

module.exports = {
  plugins: ['@vuepress/pwa']
}

该插件使用workbox-build来生成Service Worker文件，并通过register-service-worker来注册Service Worker

提示

要使您的网站完全符合 PWA 标准，您需要：

  • 在 中提供 web 应用清单和图标。.vuepress/public
  • 在中添加正确的头部链接 (opens new window)（请参阅下面的示例）。.vuepress/config.js

有关更多详细信息，请参阅有关 Web 应用清单的 MDN 文档 （打开新窗口） (opens new window).

可以访问[Favicon Generator](Favicon Generator for perfect icons on all browsers (realfavicongenerator.net) (opens new window))来直接生成所需图标和配置文件mainfest。

以下是使用 VuePress 完全符合 PWA 标准的配置示例：

module.exports = {
  head: [
    ['link', { rel: 'icon', href: '/logo.png' }],
    ['link', { rel: 'icon', href: 'favicon.ico', type: 'image/x-icon' }],
    ['link', { rel: 'manifest', href: '/manifest.json' }],
    ['meta', { name: 'theme-color', content: '#3eaf7c' }],
    ['meta', { name: 'apple-mobile-web-app-capable', content: 'yes' }],
    ['meta', { name: 'apple-mobile-web-app-status-bar-style', content: 'black' }],
    ['link', { rel: 'apple-touch-icon', href: '/icons/apple-touch-icon-152x152.png' }],
    ['link', { rel: 'mask-icon', href: '/icons/safari-pinned-tab.svg', color: '#3eaf7c' }],
    ['meta', { name: 'msapplication-TileImage', content: '/icons/msapplication-icon-144x144.png' }],
    ['meta', { name: 'msapplication-TileColor', content: '#000000' }]
  ],
  plugins: [
    [
      '@vuepress/pwa',
      {
        serviceWorker: true,
        updatePopup: true
      }
    ]
  ],
}

# # (opens new window)选项

# # (opens new window)服务员工

  • 类型：boolean
  • 违约：true

如果设置为 ，VuePress 将自动生成并注册一个true服务工作者 （打开新窗口） (opens new window)缓存内容以供脱机使用（仅在生产环境中启用）。

有一个别名模块模块，该模块还将发出以下事件：@sw-event

  • sw-ready
  • sw-cached
  • sw-updated
  • sw-offline
  • sw-error

提示

由于您只能在 HTTP URL 下注册服务工作线程，因此请确保在启用此选项之前可以使用 SSL 部署站点。

# # (opens new window)生成SWConfig

  • 类型：object
  • 违约：{}

生成SW 配置 （打开新窗口） (opens new window)的工作区构建。

# # (opens new window)更新弹出窗口

  • 类型：boolean|object
  • 违约：undefined

类型的定义如下：popupConfig

interface normalPopupConfig {
  message: string; // defaults to 'New content is available.'
  buttonText: string; // defaults to 'Refresh'
}
interface localedPopupConfig {
  [localePath: string]: normalPopupConfig
}
type popupConfig = normalPopupConfig | localedPopupConfig

此选项使弹出窗口能够刷新内容。当站点更新（即服务工作线程更新）时，将显示弹出窗口。它提供了允许用户立即刷新内容的按钮。refresh

如果没有该按钮，则新服务工作线程将处于活动状态refresh客户 （打开新窗口） (opens new window)已关闭。这意味着访问者在关闭您网站的所有选项卡之前无法看到新内容。但该按钮会立即激活新的服务工作线程。refresh

  • Type: string
  • Default: undefined

# # (opens new window)Customize the UI of SW-Update Popup

The default sw-update popup component provides a default slot which gives you the ability to fully control the appearance of the popup.

First, you need to create a global component (e.g. ) at . A simple component created based on the default component is as follows:MySWUpdatePopup``.vuepress/components

<template>
  <SWUpdatePopup v-slot="{ enabled, reload, message, buttonText }">
    <div
      v-if="enabled"
      class="my-sw-update-popup">
      {{ message }}<br>
      <button @click="reload">{{ buttonText }}</button>
    </div>
  </SWUpdatePopup>
</template>
<script>
import SWUpdatePopup from '@vuepress/plugin-pwa/lib/SWUpdatePopup.vue'
export default {
  components: { SWUpdatePopup }
}
</script>
<style>
.my-sw-update-popup {
  text-align: right;
  position: fixed;
  bottom: 20px;
  right: 20px;
  background-color: #fff;
  font-size: 20px;
  padding: 10px;
  border: 5px solid #3eaf7c;
}
.my-sw-update-popup button {
  border: 1px solid #fefefe;
}
</style>

然后，更新您的插件配置：

module.exports = {
   plugins: {
    '@vuepress/pwa': {
       serviceWorker: true,
+      popupComponent: 'MySWUpdatePopup',
       updatePopup: true
     }
  }
}
