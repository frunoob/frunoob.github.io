# immutability|react

底层数据不变性。
我们可以直接对原数据进行修改，也可以生成原数据的副本，对副本进行修改，最后将整个副本覆盖原数据，那么，这么做的意义是？

首先我们要知道，每当修改属性state时（前提是使用useState函数声明并初始化属性），react都是重新渲染该属性所在组件。

react的setState是异步的，在当前已经执行的代码里面可能不会立即生效，虽然渲染视图了，但是代码里调用这个变量，他仍然是原来的值。

这时，就要创建副本来记录值的变化，并能够被获取得到。

# nextjs

最新版本的next框架不再使用pages作为路由目录，而是改用app路由 参考 (opens new window)

layout.tsx相当于模板，所有页面都建立在该模板之上，由next框架自动生成，而page.tsx则是next框架所必须的文件，用于展示首页，就是通过 http://localhost:3000/ 连接会直接打开page.tsx这个文件，如果没有page.tsx这个文件，在执行 npm run dev这个命令的时候，就会自动生成该文件。

next不再通过 文件名字作为路径的访问地址，而是page所在文件夹的名字，例如 想要打开以下路径的文件的内容 app/page.tsx，就需要访问地址 localhost:3000/, 再比如，想要新建一个路由叫做 /test/,那就需要在app目录下新建一个目录，名字叫做test，然后在test目录下创建一个名为page.tsx的文件，注意page.tsx必须叫page不能叫别的名字，不然无效。

想要了解更多有关路由的信息 https://nextjs.org/docs/app/building-your-application/routing/defining-routes

# Detect the color scheme

如果要项目自动切换主题颜色，需要增加一些配置 (opens new window)

import { Appearance, useColorScheme } from 'react-native';
function MyComponent() {
  let colorScheme = useColorScheme();
  if (colorScheme === 'dark') {
    // render some dark thing
  } else {
    // render some light thing
  }
}
const colorScheme = Appearance.getColorScheme();
if (colorScheme === 'dark') {
  // Use dark color scheme
}

useColorScheme：当项目的主题改变时，会重新渲染页面

Appearance: 当项目主题改变时，不会重新渲染页面

# Animation

npx expo install react-native-reanimated

After the installation completes, add the Babel plugin to babel.config.js:

module.exports = function (api) {
  api.cache(true);
  return {
    presets: ['babel-preset-expo'],
    plugins: ['react-native-reanimated/plugin'],
  };
};

after that, clear bundle cache use command like npx expo start --clear

WARNING

If you load other Babel plugins, the Reanimated plugin has to be the last item in the plugins array.

for web, install the yarn add --dev @babel/plugin-transform-export-namespace-from

and add it to babel.config.js

plugins: [
  '@babel/plugin-proposal-export-namespace-from',
  'react-native-reanimated/plugin',
],

don't forget to clear cache, use command npx expo start --clear
