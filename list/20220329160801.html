# 基本操作

sudo service mysql start

# 表操作

基本语法

CREATE [TEMPORAR] TABLE [DATABASE.]table_name (
column_name1 <data_types> [列约束，多个约束使用空格隔开]，  
column_name2 <data_types> [列约束],  
...  
...
)[表选项];

创建一个图书表

CREATE TABLE `book` (
  `id` int(4) NOT NULL AUTO_INCREMENT,
  `tile` varchar(30) NOT NULL,
  `price` decimal(5,2) DEFAULT NULL,
  `description` varchar(100) NOT NULL DEFAULT '暂无简介',
  PRIMARY KEY (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8 
// 表结构如下
MariaDB [mall]> desc book;

cloumn字段
每个字段必须有数据类型。

最后一个字段后不能有逗号。

temporary临时表，会话结束后自动消失。

字段的定义： 字段名 数据类型 [NOT NULL | NULL] [DEFAULT default_value] [AUTO_INCREMENT] [UNIQUE [KEY]] [[PRIMARY] KEY] [COMMENT 'string']

表选项
1.字符集：CHARSET = utf8

2.存储引擎：ENGINE = innodb

存储引擎

常见的存储引擎：InnoDB、MyISAM、Merge、CSV、Archive等.
不同的引擎保存表的结构和数据采用不用的方式：

  • MyISAM：.frm表结构定义， .MYD表数据，.MYI表索引
  • InnoDB：.frm表结构定义, .ibd表数据、索引、表内数据字典信息, .opt表字符编码和字符排序规则
    SHOW ENGINES; // 显示存储引擎
    SHOW ENGINE engine_name {LOGS|STATUS} // 显示引擎的日志和状态

3.自增起始数： AUTO_INCREMENT = 行数
4.数据文件目录：DIRECTORY = 目录
5.索引文件目录：INDEX DERECTORY = ‘目录’
6.注释：COMMENT = 'string'
7.分区选项：PARTITION BY ...
