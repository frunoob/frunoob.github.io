# jdk动态代理

只能代理实现了接口的类。

//定义发送消息的接口
interface SmsService{
  void send(String s);
}
// 实现以上接口的类
class Sms implement SmsService {
  @Override
  void send(String s){sout(s)};
}
// 代理类 继承InvocationHandler必须实现invoke
class Proxy implements InvocationHandler{
  // 被代理的对象
  private final Object target;
  //初始化target
  Proxy(Object target){
    this.target  = target;
  }
  @Override
  public Object invoke(Object proxy,Method method,Object []args) throws Throwable{
    // 被代理方法之前执行
    // ....
    // 调用被代理类的方法
    Object result = method.invoke(target,args);
    // 被代理类方法之后执行的代码
    // 略
    return result;
  }
}
// 工厂类
class ProxyFactory{
  public static Object getProxy(Object target){
    return java.lang.reflect.Proxy.newProxyInstance(
      target.getClass().getClassLoader();
      target.getClass().getInterfaces();
      new Proxy(target);
    );
  }
}
public class Main{
  public static void main(String[]args){
    SmsService sm = (SmsService) ProxyFactory.getProxy(new Sms());
    sm.send("hello");
  }
}

# CGLIB动态代理

可代理普通类

// 定义一般类
class Sms{
  public void send(String s){
    System.out.println(s);
  }
// 定义方法拦截器
class MethodIntercept implement MethodIntercept{
  @Override
  public Object Intercept(Object o,Method method,Object [] args,MethodProxy methodProxy){
    //这里可以添加代码以在被代理方法之前添加代码
    //调用被代理类的方法
    Object result = methodProxy.invoke(o,args);
    //这里可以添加代码以在被代理方法之后添加代码
    return result
  }
}
//创建代理类
class ProxyFactory{
  public static Object getProxy(class<?> clazz){
    Enhancer enhancer = new Enhancer();
    enhancer.setClassLoader(clazz.getClassLoader);
    enhancer.setSuperClass(clazz);
    enhancer.callBack(new MethodInercept());
    return enhancer.create();
  }
}
// 使用
public class Main{
  public static void main(String[]args){
    Sms  sms = (Sms) ProxyFactory.getProxy(Sms.class);
    sms.send("hello");
  }
}

# 静态代理和动态代理的区别

静态代理是在编译的过程中讲接口类、实现类、代理类都编译出其对应的类字节码文件。而动态代理则是在运行的过程中动态的生成类字节码文件并加载到JVM中。
