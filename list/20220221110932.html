# 数据库MYSQL基础知识

# 范式

# 码(候选码)

一个表中可以唯一决定一个元组的属性的集合。例如(id, 课程名)可以确定唯一的(学生姓名，学院，该课程成绩)这一条数据，那么(id, 课程名)就是码。

# 非主属性

不是主属性就是非主属性

# 函数依赖

若x确定，y确定，则称为y函数依赖于x

# 完全函数依赖

前提是x->y（y依赖于x），若x的任意真子集都不能唯一确定一个y，那么y完全依赖于x（ ）

# 传递依赖

例如 id 姓名 系名， $系名 \stackrel{F}{\longrightarrow} 姓名 \stackrel{F}{\longrightarrow} id $ 系名依赖于姓名，姓名又依赖于id，所以系名间接依赖于id，这就是传递依赖。

# 第一范式

不重复，原子性

# 第二范式

判断是否符合2NF，就是看数据表中是否存在非主属性对于码的部分函数依赖。

若码为（id, 课程） 非主属性为 姓名，年龄。其中id能对应唯一的一个姓名，那么这就是姓名部份依赖与id。

# 第三范式

在2NF的基础上看是不是有传递依赖。

缺点是表太多影响查询速度。

# CRUD

CRUD说的就是增查改删C：就是创建（Create），R：就是查找（Retrieve），U：就是更改（Update），D：就是删除（Delete）
