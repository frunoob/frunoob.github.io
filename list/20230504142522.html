# 第二章：高并发IO的底层原理

# 2.1 IO读写的基本

为了保证内核安全，操作系统将内存分为两个部分，用户空间和内核空间。内核程序运行在内核空间，对应的进程处于内核态。用户程序运行在用户空间，对应进程处于用户态。

操作系统的核心是内核程序，它有权限访问内核空间，也能访问硬件。

用户程序如果想要访问内核空间中的数据，需要将进程状态转换为内核态。

内核态进程可以执行任意命令，调用系统的一切资源。 如果用户进程想要访问系统资源，必须通过系统调用。

缓冲区是为了解决cpu和IO之间速度不匹配，减少cpu中断而存在的。

# 2.1.1 内核缓冲区和进程缓冲区

当IO操作将内核缓冲区装满后，由系统判断何时中断设备，不需要应用操心。 上层应用使用read调用时，将数据从内核缓冲区复制到进程的缓冲区。

# 2.2 四种主要的IO模型

四种常见IO模型

阻塞和非阻塞

阻塞IO是在内核IO操作彻底完成才返回用户空间执行用户程序的操作指令。 java中创建的socket都属于阻塞IO模型。

1.同步阻塞IO

用户空间中的线程发起IO请求, 然后该线程进入阻塞状态，等内核进程彻底完成IO操作后，线程恢复。

2.同步非阻塞IO

用户空间的程序不需要等待内核缓冲区准备数据，可以立即返回用户空间去执行后续的指令。通过不断的发起系统调用，直到内核缓冲区数据准备完成。

非阻塞会占用大量cpu时间，效率低下。

3.IO多路复用

通过注册器统一管理IO请求，注册器循环查询准备就绪的线程，汇总成表返回到用户空间

4.异步IO

系统调用的过程中，用户程序不阻塞，当资源复制到用户空间之后，通知用户处理，全程用户进程都不会阻塞。
理论上 异步IO的吞吐量高于IO多路复用模型的吞吐量。window下通过IOCP实现了真正的异步IO
linux2.6版本以后才有的aio,jdk对其支持并不完善，性能提升不大

# 2.3 通过合理配置支持百万并发连接

在linux中，文件句柄默认1024,即一个进程最多可以接受1024个socket连接，这是远远不够的。
文件句柄也称文件描述符，在Linux系统中，文件描述符是内核为了高校管理已经被打开的问i按所创建的索引，是一个非负数，用于指代被打开的文件。所有的IO系统调用都是通过文件描述符来完成的。

# 查看文件句柄限制 
ulimit -n   
# 设置文件句柄数量，仅对当前用户的当前shell界面生效  
ulimit -n 1000000
# 编辑 /etc/rc.local 
ulimit -SHn 1000000
# 编辑/etc/security/limits.conf来解除系统最大文件打开数量的限制,
# 在生产环境中使用netty或者是分布式搜索引擎ElasticSearch必须要设置这两项，软性极限（超过则警告）硬性极限无法超过
soft nofile 1000000
hard  nofile 1000000

# 第三章 NIO核心

背景了解

1.4版本之前，java IO类库是阻塞IO,从1.4版本之后Java变成了异步IO。 java的NIO属于IO多路复用模型。
OIO(Old IO):阻塞式Java IO

# 3.1.1 NIO和OIO的对比

  1. OIO面向流 NIO面向缓冲区的 流就是只能按顺序读取，不能从指定位置开始读取 缓冲区就是从通道读取数据到缓冲区，或将数据从缓冲区写入到通道中。NIO不像OIO是顺序读取的，他可以从任意位置读取

  2. OIO是阻塞的，NIO是非阻塞
    在NIO模式中，调用read方法无论有没有数据都会立即返回，这是因为NIO使用了通道的多路复用及技术

  3. NIO有选择器需要系统支持 OIO没有选择器不需要系统支持

# 3.1.2 通道

在OIO中，同一个网络连接会关联到两个流： 一个是输入流一个是输出流。
在NIO中，一个网络里用一个通道表示，所有NIO的IO操作都通过连接通道完成的，通道其实就是将读写操作封装起来的结合体，既可以读取又可以操作。

# 3.1.3 选择器

通过选择器一个线程可以查询到多个通道的IO事件的就绪状态。
有了选择器可以大大减少系统的开销，因为如果没有选择器，没有连接就需要单独一个线程来查询当前连接是否已经准备就绪

# 3.1.4 缓冲区

缓冲区是非阻塞的重要前提。
所谓的通道的读取就是从通道读取到缓冲区中，通道的写入就是从缓冲区写入到通道中。

# 3.2.1 Buffer类

NIO的Buffer本质上是一个内存块，既可以读取又可以写入。 在java NIO中最常见的有八种ByteBuffer、CharBuffer、DoubleBuffer、 FloatBuffer、IntBuffer、LongBuffer、ShortBuffer、 MappedByteBuffer，前7种Buffer类型覆盖了能在IO中传输的所有Java 基本数据类型，第8种类型是一种专门用于内存映射的ByteBuffer类 型。不同的Buffer子类可以操作的数据类型能够通过名称进行判断， 比如IntBuffer只能操作Integer类型的对象。

# 3.2.2 Buffer类的重要属性

Buffer的子类会拥有一块内存，作为读写缓冲区，但是读写缓冲区并没有定义在基类中，而是定义在子类中。
