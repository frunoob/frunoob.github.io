# 超时时间|Timeout|OkhttpClient

# readTimeout

服务器端 tcp socket响应时间+IO读取时间， 通俗上讲就是

从请求到服务器响应第一个字节开始 称之为响应时间。从第一个字节到最后一个字节叫做客户算读操作总用时。

# connectTimeout

客户端请求之前，要连接到服务器，进行tcp三次握手和TLS四次握手获得钥匙。

TTFB

TTFB 是一个衡量对资源的请求和响应的第一个字节开始和到达之间时间的指标。 TTFB 是下列请求节点的时间损耗汇总：

Redirect time 重定向时延 Service worker 启动时延（如果适用） DNS 查询时延 建立连接和 TLS 所消耗时延 请求，直到响应的第一个字节到达为止的时延 减少连接建立和后端服务的时延将有助于降低 TTFB
