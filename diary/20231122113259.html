# 模拟飞腾D2000+银河麒麟V10部署java环境

模拟设备及目标设备

原始设备： CPU: AMD Ryzen 7 4800H with Radeon Graphics 2.90 GHz RAM: 12GB

模拟目标设备： CPU: Cortex-A57(ARMv8-a) 8核心 8线程 （飞腾D2000兼容ARMv8-a指令集架构） RAM: 8GB

模拟系统： 银河麒麟v10桌面版

关于SSL

当mysql开启ssl功能时（usessl=true），如果java项目不支持国密，将无法启动项目。

关于软件源

飞腾版无法修改软件源，即使强行修改也会报依赖错误并且系统会自动重置源。

如果发现软件管理工具无法搜索到 mysql jdk rides等开源工具，那么可以直接找麒麟的技术支持，提供系统序列码，很有可能是软件源的问题，没有响应的访问权限所导致的

询问客服如何安装java ，得到命令sudo apt install openjdk-8-jdk,执行之后jdk安装成功且源恢复正常，常见软件都可以搜到了

1. 使用qemu配置模拟环境 windows平台可以直接下载编译好的二进制文件，linux下需要下载源码自行编译。

2. 在qemu模拟器中安装麒麟系统 麒麟系统直接在官方申请下载即可

我下载的是 麒麟V10 桌面版 飞腾版本和麒麟V10 服务器飞腾版 （其实就是ARM64版本，一模一样）。桌面版可以正常安装，但是服务器版本在安装界面卡在右上角有个光标 就不继续了。

准备好下载好用的麒麟镜像 重命名为 kylin.iso

下载EFI引导 (opens new window) ,下载完和kylin.iso放一起

最后在当前目录下打开命令行，执行如下命令

qemu-img create -f qcow2 armrootfs.qcow2 20G 
qemu-system-aarch64 -m 4096 -cpu cortex-a72 -smp 4,cores=4,threads=1,sockets=1 -M virt -bios QEMU_EFI.fd -net nic,model=pcnet -device nec-usb-xhci -device usb-kbd -device usb-mouse -device VGA -drive if=none,file=./kylin.iso,id=cdrom,media=cdrom -device virtio-scsi-device -device scsi-cd,drive=cdrom -drive if=none,file=armrootfs.qcow2,id=hd0 -device virtio-blk-device,drive=hd0

之后会出现图形化界面，进入到系统安装界面，按照流程安装系统就行，之后启动系统的时候执行如下命令

注意，执行下面的启动系统命令之前

不配置的话启动系统会报错 找不到tap0

步骤： 需要下载windows-tap 创建一个网络适配器并重命名为tap0，并将本地网络共享给适配器tap0，同时qemu启动命令也需要改变，具体操作必应搜索 "qemu tap-windows"


qemu-system-aarch64 -m 8192 -cpu cortex-a57 -smp 8,cores=4,threads=2,sockets=1 -bios QEMU_EFI.fd -net nic,model=pcnet   -net tap,ifname=tap0  -device nec-usb-xhci -device usb-kbd -device usb-mouse -device VGA   -device virtio-scsi-device  -drive if=none,file=armrootfs.qcow2,id=hd0 -device virtio-blk-device,drive=hd0 -machine type=virt

需要注意的是 上述命令中 参数 -m 8192中8192代表的是模拟器的系统内存，单位是MB, -smp 8,cores=4,threads=2,sockets=1中 smp = coresthreadssockets, cores是核心数，threads是每个核心的线程数，sockets是cpu个数

WARNING

windows平台使用qemu模拟arm下安装kylinv10 只能安装桌面版不能安装服务器版
而桌面版和服务器版本完全不一样（一个基于ubuntu 一个基于centos）

所以接下来选择 在vmware中线安装AMD64版本的kylin-server v10系统，然后在该系统中 使用qemu模拟arm64安装arm64版本的kylin-server v10 假设已经在windows上的vmware中安装好了kylin-server,接下来进行如下操作

询问客服之后，对方说系统应该自带java，后来向技术服务人员提供了系统序列号之后，神奇的事情发生了，能搜到jdk了。所以我感觉是源的问题，没有激活系统导致无法访问官方源

1.先将系统更新到最新

提示

执行前确保以root用户身份执行

yum update && yum upgrade

2.根据官方文档安装qemu

https://wiki.qemu.org/Hosts/Linux

出现的问题

python版本过低：安装python3.8来解决该问题

参照官方文档 https://docs.python.org/3.8/using/unix.html#getting-and-installing-the-latest-version-of-python

可以不安装python，只在kylin上编译出可执行文件pyhton即可，然后继续qemu教程中的 configure --enable-debug命令，和官方不同的是，这次在结尾加上参数指定python执行文件的位置 configure --enable-debug --python=path/to/python

然后又出错了 gcc版本过低，需要7.4版本才能继续编译qemu

最终解决方案 直接用qemu历史版本5.0版本，（python和gcc都不用更新了）
