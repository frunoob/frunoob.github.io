# Java并发编程面试题

# 什么是进程？什么是线程？

查看答案

进程是系统资源分配的基本单位，线程是处理器任务调度的基本单位。

一个进程有多个线程。

# 进程与线程的区别？

查看答案

进程是系统资源分配的基本单位，线程是处理器任务调度的基本单位。

一个进程有多个线程。

内存分配：进程之间的地址空间和资源是相互独立的，线程之间是共享的。

切换时资源开销：进程切换资源开销大，线程资源开销小。

# 并行与并发的区别？

查看答案

并发：一个处理器处理多个任务，每个任务按照时间片进行轮转执行。

并行：多个处理器同时处理多个任务。

# 多线程的优缺点？

查看答案

优点：充分利用多核CPU的优势。

缺点：1.死锁 2. 上下文切换频繁影响多线程的执行速度。

# 用户线程和守护线程的区别？

查看答案

调用Thread.setDaemon(bool on)，true就是将线程设置为守护线程，false是用户线程。

一般创建的线程都是用户线程，守护线程是用来服务于用户线程的例如垃圾回收线程。

区别：

用户线程存在java虚拟机不会结束，仅有守护线程的话java虚拟机会结束。

# 线程死锁是如何产生的，如何避免？

查看答案

必要条件：

  1. 互斥条件
  2. 请求和保持条件
  3. 不剥夺条件
  4. 循环等待条件

解决：

  1. 一次性获得全部资源
  2. 申请不到资源就主动放弃所持有的资源
  3. 顺序执行所有线程

# 线程的周期和状态？

查看答案

初始状态，运行状态（就绪状态、运行中状态）、阻塞状态、等待状态、超时等待状态、终止状态。

周期：new一个线程之后，线程进入初始状态，在调用start方法后，线程进入运行状态中的就绪状态，如果获得到cpu就进入到运行中状态，在这个过程中，线程可以主动进入到等待状态直到被其他线程唤醒，或者被动进入到阻塞状态直到获得锁，也可以等待指定时间进入到超时等待状态，如果run方法执行完毕线程就会进入到终止状态。

# Runnable和Callable的区别？

查看答案

相同点：

  • 都是接口
  • 都需要调用start启动线程

不同点：

  • Runnable核心是run方法无返回值，Callable核心是call方法允许返回值。
  • call()可抛出异常，run()不可以。

# 线程的run()和start()区别？

查看答案
  • run里面是线程要执行的代码，start是用来启动 一个线程的。
  • run可以重复调用，start只能调用一次。

# 为什么要先调用start()，而不直接调用run()？

查看答案

run不会创建一个线程，run()里面是线程要执行的代码，start()才是创建线程。

# 线程的sleep()方法和yield()方法有什么不同？

查看答案
  • sleep()方法是使线程暂停指定的时间，不消耗CPU时间片。
  • sleep()方法会进入到阻塞状态，yield()会进入到就绪状态。
  • sleep()方法会抛出中断异常，而yield()方法无需抛出异常。

# sleep()方法和wait()方法的区别？

查看答案

相同点：

  • 二者都会使线程进入到阻塞状态。
  • wait和sleep都是可中断方法，被中断后都会收到中断异常。

不同点：

  • wait是Object的方法，sleep是Thread方法。
  • wait必须在同步方法中，sleep不需要。
  • sleep在短暂休眠后会主动退出阻塞，wait需要其他线程中断才能退出阻塞。
  • 线程执行sleep方法不会释放锁，wait方法会释放锁。

sleep 方法可以让线程进入 waiting 状态，并且不占用 cpu 资源，但是不会释放锁，直到规定时间后再执行，休眠期间如果被中断，会抛出异常并清除中断状态

# 为什么wait()、notify()、notifyAll()、被定义在Object类中而不是在Thread类中。

查看答案

显的原因是JAVA提供的锁是对象级的而不是线程级的，每个对象都有锁，通过线程获得。如果线程需 要等待某些锁那么调用对象中的wait()方法就有意义了。如果wait()方法定义在Thread类中，线程正在 等待的是哪个锁就不明显了。简单的说，由于wait，notify和notifyAll都是锁级别的操作，所以把他们 定义在Object类中因为锁属于对象 。

# 为什么wait()、notify()、notifyAll()在同步方法或同步块中调用？

查看答案
  1. 只有在调用线程拥有某个对象的独占锁时，才能够调用该对象的wait(),notify()和notifyAll()方法。
  2. 如果你不这么做，你的代码会抛出IllegalMonitorStateException（非法锁状态异常）异常。
  3. 还有一个原因是为了避免wait和notify之间产生竞态条件。

# 为什么Thread类中的sleep()和yield()方法是静态的？

查看答案

这二者都是需要正在执行的线程自身调用的，如果让别的线程调用是无意义的，所以是静态方法。

# java如何实现两个线程之间的通信和协作？

查看答案

在同步代码块或方法中使用Object类的wait()、notify()、notifyAll()。

# 同步方法和同步块哪个效果好？

查看答案

同步块好，它的锁的范围更灵活。

同步块在需要锁住的代码块中锁住相应的对象，同步方法锁住整个对象。

# 线程过多会造成什么异常？

查看答案

# 什么是ThreadLocal？

查看答案

本地线程变量，ThreadLocal会为变量在每个线程创建一个副本，只能在线程内部访问副本变量，对于其他线程来说是封闭的。

# 什么是线程池？

查看答案

线程池就是一组线程，有任务提交的时候就分配线程，没有任务的时候线程池中的线程就处于等待状态，线程池的作用是为了避免创建和销毁线程时带来的时间消耗。

# 为什么使用线程池？

查看答案

降低资源消耗，重复利用已创建的线程来避免反复创建和释放线程所带来的资源消耗

提高响应速度，无需创建线程只要在线程池中申请即可立即执行

方便线程的统一管理，避免线程的分配不均
