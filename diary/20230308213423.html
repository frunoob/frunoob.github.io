# 单片机概念及其特点

单片机得英文叫法： MCU: Micro-Controller Unite 微控制器
SCM: Single Chip Microcomputer 单片机

计算机系统应该包含得基本部分： cpu, ROM(预先写入得程序),RAM数据存储器相当于内存，定时器/计数器（产生延迟），输入输出接口，外部通信接口，中断系统

51单片机的存储结构分为片内存储和片外存储，不同型号的51单片机的片内存储的容量不同，片外存储空间为64KB，通过外部总线连接。

外部总线 宽度为16 地址范围是 0000H ~ FFFFH 寻址能力是 64ki 单个地址所占空间为 1 Byte

入栈： Stack Pointer堆栈指针先加一，指向下一个单元，然后再将数据存放当当前指针所指向的单元。
出栈： Stack pointer先取出当前单元的数据，然后将指针内容减一
由低地址向高地址方向入站，由高地址向低地址方向出栈
51单片机入栈数据和出栈数据的单位是字节。

51单片机的1脚位置有个圆点标志，顺着1脚依次增加，分别是2、3、4、5脚，等等。

P0口（39~32脚）：P0.0~P0.7 统称为P0口。在不接片外存储器与不扩展I/O口时，作为双向输入/输出口。在扩展了片外存储器与I/O扩展时，P0口时分复用为低8为地址总线和双向数据总线。 （低八位：在16位系统中，二进制数从右往左数1-8位，高八位即从右往左数第9至16位）

p1口（1-8脚）： P1.0-p1.7统称为P1口，可作为双向I/O口使用。

P2口（21-28脚）： 一般可作为双向I/O口使用，在扩展片外存储器时，P2口作为高8位地址总线。

# 数据类型与存储类型

除了和C语言一样的数据类型之外，C51增加了4中扩展数据类型

  • bit: bit可以是0或是1(true)
  • 特殊功能寄存器sfr 8051的特殊功能寄存器分布在片内数据存储区的地址单元80H~FFH之间，sfr数据类型占用一个内存单元。利用他可以访问单片机内所有的特殊寄存器。
  • 特殊功能寄存器sfr16 srf16数据类型占用两个内存单元。
  • 特殊功能位 sbit sbit是8051片内特殊功能寄存器的可寻址位。

数据存储类型
低128字节可直接寻址，高128字节(80H~FFH)只能间接寻址，从地址20H开始的16字节可位寻址。内部数据存储区可分为3个不同的数据存储类型：data，idata，bdata
访问片外数据存储区比访问片内存储区要慢，因为片外数据存储区需要通过数据指针加载地址来间接寻址访问。
C51提供了两种不同的数据存储类型xdata和pdata来访问片外数据存储区。
程序存储区只能读不能写，C51提供了code存储类型来访问程序存储区。

各个存储区说明
（1）DATA区 该区寻址速度最快，应把经常使用的变量存放在DATA区，但是DATA区的存储空间是有限的。DATA区除了程序变量之外，还包含了堆栈和寄存器组。DATA区声名中的存储类型标识符为data，通常指片内RAM的128字节的内部数据存储变量，可直接寻址。

WARNING

标准变量和用户自声名变量都可存储在DATA区中，C51语言使用默认的寄存器组来传递参数，这样data区至少失去了8字节空间。内部堆栈溢出时，程序会意外复位，所以要时刻保持有较多的堆栈空间。

（2）BDATA区实质上时DATA中的位寻址区，在这个区声名的变量就可以直接进行位寻址，标识符时bdata，指的是片内RAM可位寻址的16字节存储区（字节地址为20H~2FH）中的128位。 (2FH-20H+1H ) * 8H = 128位

WARNING

BDATA区不允许声名float和double型变量。

（3） IDATA区使用寄存器作为指针来进行间接寻址。存放使用比较频繁的变量。idata声名中的存储类型标识符为idata，指的是片内RAM的256字节的存储区，只能间接寻址，速度比直接寻址慢。
（4） PDATA区和XDATA区 PDATA区和XDATA区位于片外存储区，标识符分别为pdata和xdata，
（5）程序存储区CODE，标识符code，存储的数据是不可改变的。

值域

data,idata,pdate(0~255)
bdata(0\1)
xdata,code(0~65535)
