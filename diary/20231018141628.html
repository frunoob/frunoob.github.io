# problem|mvnw|Warning: JAVA_HOME environment variable is not set|mvnw打包失败|package|compile|找不到符号

问题描述

使用 mvnwm命令时，提示java环境变量没有设置，但是在命令明明可以正常打印JAVA_HOME

问题一
好像是因为使用了sudo命令导致上下文环境发生了改变。 所以mvnw命令找不到java_home，去掉sudo再试试就好了，因为本身就是root身份

问题二
首次执行命令时，速度奇慢：因为首次执行需要下载指定版本的maven，而wrapper中配置文件默认的镜像源是国外的所以速度奇慢，修改为国内镜像源就可以了

打开./mvn/wrapper/maven-wrapper.properties 修改第一行为如下内容即可

# distributionUrl=https://repo.maven.apache.org/maven2/org/apache/maven/apache-maven/3.9.3/apache-maven-3.9.3-bin.zip
distributionUrl=https://mirrors.cloud.tencent.com/apache/maven/maven-3/3.9.5/binaries/apache-maven-3.9.5-bin.zip
wrapperUrl=https://repo.maven.apache.org/maven2/org/apache/maven/wrapper/maven-wrapper/3.2.0/maven-wrapper-3.2.0.jar

问题三：打包命令失败 mvnw package

找不到符号报错，这个命令总是有这个问题不知道为啥，按理说应该可以正常使用

~换个命令就好了 ./mvnw jar:jar -f ./pom.xml~

前提是pom文件里配置了maven-compiler-plugin

	<build>
		<plugins>
		<plugin>
            <groupId>org.apache.maven.plugins</groupId>
            <artifactId>maven-compiler-plugin</artifactId>
            <version>3.8.1</version>
            <configuration>
                <source>17</source> <!-- java版本号，取决于你的当前项目使用的java版本 -->
                <target>17</target> <!-- java版本号，取决于你的当前项目使用的java版本 -->
            </configuration>
        </plugin>
		
		</plugins>
	</build>

换命令打出来的jar包，没法执行，显示找不到主类入口

最终发现是因为lombok注解识别不出来，导致找不到符号，所以修改pom文件，在 annotationProcessor 里面增加了lombok 注解信息，具体代码如下所示

<plugin>
				<groupId>org.apache.maven.plugins</groupId>
				<artifactId>maven-compiler-plugin</artifactId>
				<configuration>
					<annotationProcessorPaths>
						
						<!-- other annotation processors -->
						<path>
							<groupId>org.projectlombok</groupId>
							<artifactId>lombok</artifactId>
							<version>1.18.30</version>
						</path>
					</annotationProcessorPaths>
				</configuration>
			</plugin>

问题解决！
