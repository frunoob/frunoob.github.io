# 范式

# 元组

行为元组，列为属性。

# 码

对应列能唯一表示实体的属性

# 候选码

能够唯一标识一个元组的某一属性或属性组，且任何子集都不能再标识。

# 主码（主键）

从候选码中选出的，一个实体集（数据库表）只能有一个主码，但有多个候选码。

# 非主属性

不是主属性就是非主属性

# 函数依赖

若x确定，y确定，则称为y函数依赖于x

# 完全函数依赖

前提是x->y（y依赖于x），若x的任意真子集都不能唯一确定一个y，那么y完全依赖于x（ ）

# 传递依赖

例如 id 姓名 系名， 系名依赖于姓名，姓名又依赖于id，所以系名间接依赖于id，这就是传递依赖。

# 第一范式

不重复，原子性

# 第二范式

判断是否符合2NF，就是看数据表中是否存在非主属性对于码的部分函数依赖。

若码为（id, 课程） 非主属性为 姓名，年龄。其中id能对应唯一的一个姓名，那么这就是姓名部份依赖与id。

# 第三范式

在2NF的基础上看是不是有传递依赖。

缺点是表太多影响查询速度。

# CRUD

CRUD说的就是增查改删C：就是创建（Create），R：就是查找（Retrieve），U：就是更改（Update），D：就是删除（Delete）

# 关系模型

MySQL是关系型数据库，表与表之间需要建立“一对多”、“多对一”、“一对一”的关系。

在关系型数据库表中，关系通过主键和外键来维护。

# 主键

身份证号、手机号、邮箱都是业务相关的字段，即可能由于业务的变更而发生改变，所以能够作为主键的常见类型有：

  1. 自增整数（int 最多21亿条记录）（bigint 最多992亿条记录）
  2. 全局唯一GUID类型：使用全局唯一的字符串作为主键。GUID算法通过网卡MAC地址、时间戳和随机数保证计算结果的 唯一性。

一般将主键设置为

BIGINT NOT NULL AUTO_INCREMENT

# 联合主键

多个字段唯一标识记录，即两个或者多个字段都设置为主键，这种主键 被称为联合主键。

允许重复

对于联合主键允许任意一列中存在重复值 ，但所有主键列都同时重复就不行了。

id u_id other_columns
1 AAA
2 AAA
2 BBB

不允许同时重复

id u_id other_columns
1 AAA
2 AAA
2 AAA

# 外键

在A表中，添加B表中主键作为一个字段，称该字段为外键。

以下是创建外键的方法：

ALTER  TABLE A //修改A表的结构
ADD CONSTRAINT A_B_id  // 外键约束名，任意
FOREIGN KEY (class) // 外键在表A中的字段名 
REFERENCES B (id) // 将外键关联到B表的主键id

以下是删除外键（并没有删除列，只是删除了关联关系）：

AlTER TABLE  A 
DROP FOREIGN KEY  A_B_id //外键约束名

# 索引

定义： 索引是对数据库表中的一列或多列进行预排序的数据结构。

优点：提高了对记录的查找速度，不需要在整个表中进行查找。

缺点：每次进行表修改的时候都要更新索引。

注意

索引对于用户的应用程序来说是透明的。

通过唯一索引可以限制列中值不能重复

# 创建索引

ALTER TABLE student 
ADD INDEX idx_score (score) //

student为表名，idx_score 是索引名，任意;score是要创建索引的列。

# 创建唯一索引

索引列不能有重复

ALTER TABLE student
ADD UNIQUE INDEX idx_score  (score)

# 添加唯一约束但不创建索引

ALTER TABLE student
ADD CONTRIANT idx_score UNIQUE (score)

仅仅添加了“score列不能存在重复值”的约束条件，但没有创建索引。

# 排序

# 分页

LIMIT pagesize OFFSET pagesize*(pageindex-1)

缺点

由于分页之前需要知道记录的总数select count(id) from talbe;,所以当数据量大的时候就会严重影响性能。

优化

不使用count求记录总条数，而是根据业务确定pagesize，如果获取到的记录条数等于pagesize说明下一页还有记录，如果返回的记录条数小于pagesize说明到达了最后一页

# 聚合函数

# count函数

统计某一列记录的总条数。

COUNT(*)或COUNT(id)都是统计记录的总条数。没有记录返回的是零

SELECT COUNT(*) num FROM student;

结果返回一个表：

num
10

# SUM

返回某一列的总和，必须是数值类型的列。

如果是字符串类型的列则返回的是0。

SELECT SUM(score) FROM student;

语句执行结果为：

SUM(score)
923

# AVG

返回列的平均值（必须是数值类型的列）。

如果不是数值类型的列则返回0；

SELECT AVG(score) FROM student;

语句执行结果为：

AVG(score)
83.2000

# MAX和MIN

返回最大值和最小值的列表。如果是字符串列的话就返回第一个值和最后一个值。

# OUTER JOIN

外连接包含了 左连接、右连接、全连接。

注意

MySQL中没有FULL OUTER JOIN

通过 LEFT OUTER JOIN NUON RIGHT OUTER JOIN实现全连接

::: waring UNION使用注意事项

UNION 内部的每个 SELECT 语句必须拥有相同数量的列。

列也必须拥有相似的数据类型。

同时，每个 SELECT 语句中的列的顺序必须相同。

:::

SELECT 
    s.id, c.name classname, s.name, s.class_id, s.score
FROM
    students s
        LEFT OUTER JOIN
    classes c ON s.class_id = c.id 
UNION SELECT 
    s.id, c.name classname, s.name, s.class_id, s.score
FROM
    students s
        RIGHT OUTER JOIN
    classes c ON s.class_id = c.id
